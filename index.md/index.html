搭建hexo博客

发表于 2020-10-29 分类于 搭建博客

hexo搭建博客

前提

已经搭建好github主页：username.github.io

环境配置

  • hexo文档：https://hexo.io/zh-cn/docs/

  • 所需软件：

    • Git
    • hexo
    • node.js

  • 安装Git

  • 安装node.js 官网： http://nodejs.cn/ 下载：http://nodejs.cn/download/

    • 将nodejs添加到环境变量
    • CMD里面输入node+回车不报错

  • 安装hexo

    • 保证nodejs正确安装后在cmd里输入
      npm install -g hexo-cli

    • 安装完成后检查是否安装成功
      hexo -v 返回版本号则正确安装

    • 初始化hexo 在磁盘中找一个位置
      hexo init myblog 其中myblog可以随便写
      cd myblog 进入到该文件夹
      npm install 完成后初始化成功

      ls命令查看目录 此目录之后称为根目录

    • node_modules: 依赖包

    • public：存放生成的页面

    • scaffolds：生成文章的一些模板

    • source：用来存放你的文章

    • themes：主题

    • _config.yml: 博客的配置文件

本机试用官方示例

  • 在上面的根目录下打开cmd
    hexo server 默认情况下在4000端口
  • 浏览器里输入localhost:4000查看博客
  • ctrl+c关闭服务

将本机博客部署到github

前提已经有了github page

  • 在根目录下找到_config.yml，该文档为hexo的配置文件，打开

  • 在文档最后，更改代码，示例

    deploy:
type: git
repo: git@github.com:username/username.github.io.git
branch: main```

  username是自己的账户名

  • 在cmd中加载部署模块
    npm install hexo-deployer-git --save

  • 上传到github
    hexo deploy 可缩写 hexo d

Git学习

发表于 2020-10-23 分类于 Git学习

Git配置

1.git config

  1. /etc/gitconfig文件 系统中对所有用户都普遍适用的配置。若使用 git config 时用 –system 选项，读写的就是这个文件。
  2. ~/.gitconfig文件 用户目录下的配置文件只适用于该用户。若使用 git config 时用 –global 选项，读写的就是这个文件。

Git是分布式的版本控制系统，每个机器都必须自报家门：你的名字和Email地址
git config --global user.name "runoob"
git config --global user.email test@runoob.com

2.查看配置信息

git config --list

  • user.name=lleo
  • user.email=leodesksss@nit.zju.edu.cn

more ~/.gitconfig

[gui]
recentrepo = C:/Users/leo/Documents
[user]
name = lleo
email = leodesksss@nit.zju.edu.cn

创建仓库(版本库)

初始化 ：
git init
查看.git目录
ls -ah

把文件添加到版本库

  1. git add {FileName} 添加文件（单个）
    git add README.md
  2. git commit -m "comment" 提交文件（多个）
  3. git status 查看当前状态，有哪些文件被挂起待提交
  4. git diff 查看当前修改
  5. git log 以当前版本为HEAD查看之前版本号
  6. git reset --hard {commit_ID} 回到某个版本
  7. git reflog 查看之前的操作，以及所有的版本号
  • git checkout 待学习 –撤销修改

删除文件

git rm
rm
两者有区别 后者是修改 前者是从版本库中删除

远程仓库

  1. 添加公钥到github .ssh文件夹里面两个文件
  2. 改变分支名字 master –> main
    git branch -M main
  3. 连接到远程仓库
    git remote add origin https://github.com/leoDESKSSS/learngit.git
  4. 上传本地仓库到github 默认远程仓库名字为origin
    git push -u origin main
  5. 克隆远程仓库
    git clone git@github.com:leoDESKSSS/learngit2.git
