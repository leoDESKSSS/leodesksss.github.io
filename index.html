Git学习

发表于 2020-10-23 分类于 Git学习

Git配置

1.git config

  1. /etc/gitconfig文件 系统中对所有用户都普遍适用的配置。若使用 git config 时用 –system 选项，读写的就是这个文件。
  2. ~/.gitconfig文件 用户目录下的配置文件只适用于该用户。若使用 git config 时用 –global 选项，读写的就是这个文件。

Git是分布式的版本控制系统，每个机器都必须自报家门：你的名字和Email地址
git config --global user.name "runoob"
git config --global user.email test@runoob.com

2.查看配置信息

git config --list

  • user.name=lleo
  • user.email=leodesksss@nit.zju.edu.cn

more ~/.gitconfig

[gui]
recentrepo = C:/Users/leo/Documents
[user]
name = lleo
email = leodesksss@nit.zju.edu.cn

创建仓库(版本库)

初始化 ：
git init
查看.git目录
ls -ah

把文件添加到版本库

  1. git add {FileName} 添加文件（单个）
    git add README.md
  2. git commit -m "comment" 提交文件（多个）
  3. git status 查看当前状态，有哪些文件被挂起待提交
  4. git diff 查看当前修改
  5. git log 以当前版本为HEAD查看之前版本号
  6. git reset --hard {commit_ID} 回到某个版本
  7. git reflog 查看之前的操作，以及所有的版本号
  • git checkout 待学习 –撤销修改

删除文件

git rm
rm
两者有区别 后者是修改 前者是从版本库中删除

远程仓库

  1. 添加公钥到github .ssh文件夹里面两个文件
  2. 改变分支名字 master –> main
    git branch -M main
  3. 连接到远程仓库
    git remote add origin https://github.com/leoDESKSSS/learngit.git
  4. 上传本地仓库到github 默认远程仓库名字为origin
    git push -u origin main
  5. 克隆远程仓库
    git clone git@github.com:leoDESKSSS/learngit2.git
